Dependency injection

Services, providers, and shared state

Dependency Injection lets Angular provide services where they are needed while keeping components focused on the UI.

Services

Services hold reusable logic such as API calls, shared state, or business rules.

Providers

Providers tell Angular how to create and inject a dependency. You can register a provider at the component, module, or application level.

Injector hierarchy

Angular creates injectors at different levels. A service provided at a child injector can override one provided at a parent injector.

Singleton services

When a service is provided in root, Angular creates a single shared instance for the whole app.

Injection tokens

Injection tokens provide a flexible way to inject values or implementations when a class token is not enough.
