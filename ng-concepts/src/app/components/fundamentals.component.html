Angular fundamentals

Core building blocks

Angular fundamentals cover the building blocks that shape every Angular app: architecture, components, modules, templates, data binding, directives, and pipes.

Angular architecture

An Angular app is built from components, templates, services, and modules. The framework uses change detection and dependency injection to manage the UI and business logic.

Components

Components define the UI, behavior, and styling of a view. Each component has metadata, a template, and a class that controls its logic.

Modules and standalone components

NgModule groups related declarations, imports, and providers. Standalone components remove the need for NgModule for many small features and can be used directly.

Templates

Templates define the HTML structure and bind data from the component class to the UI.

Data binding

Angular supports interpolation, property binding, event binding, and two-way binding to connect the DOM and component logic.

Directives

Directives extend HTML by adding behavior. Component directives create components, structural directives change the DOM layout, and attribute directives modify the appearance or behavior of an element.

Pipes

Pipes transform values for display. Angular provides built-in pipes such as currency and date, and you can create custom pipes for project-specific formatting.
